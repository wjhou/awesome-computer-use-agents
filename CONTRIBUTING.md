# Contributing to Awesome Computer Use Agents

Thank you for your interest in contributing to this collection!

## How to Contribute

### Adding a Paper

1. Determine the appropriate category:
   - `papers/surveys/` - Survey and review papers
   - `papers/models/` - Model architecture papers
   - `papers/benchmarks/` - Benchmark and dataset papers
   - `papers/methods/` - Training methods and techniques
   - `papers/safety/` - Safety and security research

2. Add an entry with:
   - Title
   - Authors/Organization
   - Date/Venue
   - Link (arXiv, ACL Anthology, etc.)
   - Tags
   - Brief description

### Adding a Product

1. Add to `products/README.md`
2. Include:
   - Product name and company
   - Status (GA, Preview, Beta)
   - Platform (Web, Desktop, Mobile)
   - Link
   - Key features
   - Tags

### Adding a Framework

1. Add to `frameworks/README.md`
2. Include:
   - Name and organization
   - GitHub link and stars
   - Language and platform
   - Key features
   - Tags

## Tagging Convention

### Category Tags
- `survey`, `model`, `benchmark`, `dataset`, `method`, `safety`, `security`

### Platform Tags
- `web`, `desktop`, `mobile`, `android`, `ios`, `windows`, `macos`, `cross-platform`

### Technical Tags
- `vlm`, `llm`, `grounding`, `reinforcement-learning`, `multi-agent`, `on-device`

### Organization Tags
- `microsoft`, `google`, `anthropic`, `openai`, `bytedance`, `alibaba`, `tencent`

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request with a clear description

## Quality Guidelines

- Ensure links are working
- Use consistent formatting
- Include accurate dates and venues
- Provide meaningful descriptions
- Add appropriate tags

## Questions?

Open an issue if you have questions or suggestions!
